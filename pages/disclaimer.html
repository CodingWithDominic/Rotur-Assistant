Privacy Disclaimer

Data Handling

By default, user data, including the tokens associated with all the accounts added, are stored in the extension's local storage. By default, data isn't synced across devices you use this extension on. It is only synced once you press the "Sync" button in the account manager. Your data is not automatically shared with any website outside of this extension. It is only passed to a supported website once you click "Switch to selected account on service" in the account manager while on a supported website. Your data is not sold to any data brokers. For more information about how your Rotur account data is handled outside of this extension (even when it's not installed or in use), see Rotur's own privacy policy page.

Data Visibility

Your data is only used so that this extension can send and receive certain requests to the Rotur API on your behalf (upon actions that you do within this extension, not autonomously). If you opt for daily credit notifications in the wallet app, then some of your data (only non-sensitive data like names and UUIDs) is used in the background while the extension appears to not be running, in order for Rotur Assistant to know which account opted for the daily credit notification when it comes time to send the notification. Similarly, if you enable RPC in the Rotur RPC app, then the extension will use some of your data, including your account's token, to pass the RPC data and upkeep the connection to the Rotur websocket. The developers of this extension can't see your account tokens nor your passwords or password hashes. If you don't believe me, you can look around in the source code on github. The only collected data that may be visible to the developers of Rotur Assistant is the analytical data Google automatically collects for any extension submitted to the chrome web store.
